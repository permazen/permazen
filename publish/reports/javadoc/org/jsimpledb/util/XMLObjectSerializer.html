org.jsimpledb.util

Class XMLObjectSerializer

  • java.lang.Object
    • org.jsimpledb.util.AbstractXMLStreaming
      • org.jsimpledb.util.XMLObjectSerializer

  • public class XMLObjectSerializer
extends AbstractXMLStreaming
    Utility methods for serializing and deserializing objects in a Transaction to/from XML.

    There are two supported formats. The "Storage ID Format" specifies objects and fields using their storage IDs and supports all possible database contents. 

      <objects>
      <object storageId="100" id="64a971e1aef01cc8" version="1">
          <field storageId="101">George Washington</field>
          <field storageId="102">true</field>
          <field storageId="103">
              <entry>
                  <key>teeth</key>
                  <value>wooden</value>
              </entry>
          </field>
          <field storageId="104">c8b84a08e5c2b1a2</field>
      </object>
      ...
  </objects>

    "Name Format" differs from "Storage ID Format" in that object types and fields are specified by name instead of storage ID. However, it does not support schemas that use names that are not valid XML tags. 

      <objects>
      <Person id="64a971e1aef01cc8" version="1">
          <name>George Washington</name>
          <wasPresident>true</wasPresident>
          <attributes>
              <entry>
                  <key>teeth</key>
                  <value>wooden</value>
              </entry>
          </attributes>
          <spouse>c8b84a08e5c2b1a2</spouse>
      </Person>
      ...
  </objects>

    Some details on the input logic:

    • Simple fields that are equal to their default values, and complex fields that are empty, may be omitted
    • The "version" attribute may be omitted, in which case the default schema version associated with the Transaction being written to is assumed.
    • The "id" attribute may be omitted, in which case a random unassigned ID is generated
    • Any object ID (including the "id" attribute) may have the special form generated:TYPE:SUFFIX, where TYPE is the object type name or storage ID and SUFFIX is an arbitrary string. In this case, a random, unassigned object ID for type TYPE is generated on first occurrence, and on subsequent occurences the previously generated ID is recalled. This facilitates input generated via XSL and the generate-id() function. The configured GeneratedIdCache keeps track of generated IDs.

        public int read(XMLStreamReader reader)
         throws XMLStreamException
        Import objects into the Transaction associated with this instance from the given XML input. This method expects to see an opening <objects> as the next event (not counting whitespace, comments, etc.), which is then consumed up through the closing </objects> event. Therefore this tag could be part of a larger XML document.

        The input format is auto-detected for each <object> based on the presense of the "type" attribute.
