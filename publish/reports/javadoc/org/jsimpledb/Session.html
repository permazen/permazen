Class Session

  • java.lang.Object
    • org.jsimpledb.Session
  • Direct Known Subclasses:
    ParseSession

    public class Session
extends Object
    Utility class for programmatic JSimpleDB database access.

    Instances operate in one of two modes, either JSimpleDB mode or core API mode. In the latter mode, no Java model classes are required and only the core API may be used. Of course the core API can also be used in JSimpleDB mode.

    This class is not thread safe.

      • perform

        public boolean perform(Transaction tx,
              Session.Action action)
        Perform the given action within the given existing transaction, if any, otherwise within a new transaction. If tx is not null, it will used and left open when this method returns. Otherwise, if there is already an open transaction associated with this instance, or else the current thread, it will be used and left open when this method returns; otherwise, a new transaction is created for the duration of action and then committed.

        If tx is not null and there is already an open transaction associated with this instance and they are not the same transaction, an IllegalStateException is thrown.

        If action throws an Exception, it will be caught and handled by reportException() and then false returned.

        Parameters:
        tx - transaction in which to perform the action, or null to create a new one (if necessary)
        action - action to perform
        Returns:
        true if action completed successfully, false if the transaction could not be created or action threw an exception
        Throws:
        IllegalStateException - if tx conflict with the already an open transaction associated with this instance
        IllegalArgumentException - if action is null
