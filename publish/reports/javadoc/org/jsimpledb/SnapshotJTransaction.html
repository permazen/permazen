org.jsimpledb

Class SnapshotJTransaction

  • java.lang.Object
    • org.jsimpledb.JTransaction
      • org.jsimpledb.SnapshotJTransaction

  • public class SnapshotJTransaction
extends JTransaction
    An initially empty, in-memory JTransaction that persists indefinitely.

    SnapshotJTransactions hold a "snapshot" some portion of the state of a JTransaction for later use. Each SnapshotJTransactions contains its own set of "snapshot" JObjects that are copies of corresponding "real" database JObjects (as of the time they were copied). Because a SnapshotJTransaction lives in memory indefinitely, these objects can be used just like normal Java objects, outside of any regular transaction. However, in addition JSimpleDB features such as indexing, listeners, validation, etc. will continue to work normally.

    Typically, SnapshotJTransactions are not utilized directly; instead each JTransaction has an associated default SnapshotJTransaction instance, and JObject.copyOut() is invoked to copy an object there. However, more general usage is possible by creating instances of this class directly.

    This class guarantees that for each ObjId, there will only exist a single JObject instance associated with it. While there is a single pool of globally unique "database" JObjects per JSimpleDB, each SnapshotJTransaction maintains its own pool of "snapshot" JObject instances. In turn, each snapshot JObject instance is associated with a specific SnapshotJTransaction.

    • Constructor Detail

      • SnapshotJTransaction

        public SnapshotJTransaction(JTransaction jtx,
                    ValidationMode validationMode)
        Create a new instance based on the specified transaction. This new instance will be initially empty but will have the same recorded schema history as jtx.

        It is not normally necessary to create SnapshotJTransactions, as every JTransaction automatically has an associated default SnapshotJTransaction created for it and made available via JTransaction.getSnapshotTransaction().

        Parameters:
        jtx - open transaction from which to snapshot objects
        validationMode - the ValidationMode to use for this transaction
        Throws:
        IllegalArgumentException - if either parameter is null
        StaleTransactionException - if jtx is no longer usable
