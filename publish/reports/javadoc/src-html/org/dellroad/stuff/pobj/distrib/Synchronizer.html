022/**
023 * Synchronizes a {@link PersistentObject} database across multiple nodes using {@code git(1)}.
024 *
025 * <p>
026 * This class provides a simple and lightweight way to build a "master-master" multi-node database
027 * based on {@link PersistentObject} object graphs. The basic idea is to apply {@code git(1)}'s
028 * efficient communication protocol and powerful merging algorithms to the {@link PersistentObject}
029 * database's XML representation. As a result, all nodes can make simultaneous changes, updates are
030 * communicated quickly, and in only a small corner case -- two conflicting changes to the same
031 * data element applied at virtually same time -- must one node's change "win" and the other's "lose".
032 * No global communication or two-phase commit protocol is required, and recovery from arbitrary
033 * network problems is automatic.
034 * </p>
035 *
036 * <p>
037 * This class works by registering as a {@linkplain PersistentObjectListener listener} on a {@link PersistentObject}
038 * and also managing a private {@code git(1)} repository which is configured to track zero or more remotes.
039 * Each time a change occurs to the {@link PersistentObject} database, an updated XML copy of the database
040 * is committed to the {@code git(1)} repository, and then <i>synchronization</i> is performed: all remotes are fetched
041 * and their corresponding {@code git(1)} branches merged into the local {@code git(1)} branch (this process
042 * is performed by {@link #synchronize}). During synchronization, several merge strategies are tried until an XML file
043 * is produced that {@linkplain org.dellroad.stuff.pobj.PersistentObjectDelegate#validate validates}.
044 * If synchronization results in a non-trivial merge (i.e., any changes), the local {@link PersistentObject} is updated
045 * with the result and therefore made aware of the other nodes' changes. In this way, all nodes have their
046 * {@link PersistentObject} databases automatically synchronized and merged as they evolve.
047 * </p>
048 *
049 * <p>
050 * The user of this class must separately initialize each node's {@code git(1)} repository, configure the
051 * {@linkplain #setRemotes remotes}, and determine when to {@link #synchronize} remote changes:
052 * as mentioned above, synchronization is performed automatically when the local {@link PersistentObject} database changes,
053 * but for remote changes either some external notification process must trigger a call to {@link #synchronize},
054 * or {@link #synchronize} must be invoked at regular intervals, or (for robustness in the face of transient
055 * network issues) both.
056 * </p>
057 *
058 * <p>
059 * Merges are attempted using the following strategies (in this order) until there are no merge conflicts
060 * and a {@linkplain org.dellroad.stuff.pobj.PersistentObjectDelegate#validate valid} XML file is achieved:
061 *  <ul>
062 *  <li>Recursive merge using the patience algorithm</li>
063 *  <li>Same as previous, but resolving all conflicts one way or the other</li>
064 *  <li>Simply choosing one file or the other</li>
065 *  </ul>
066 * </p>
067 *
068 * <p>
069 * If conflicts occur in attempt #1, but attempt #2 or #3 succeeds and the other node is the "winner", then
070 * {@link #handleConflictOverride handleConflictOverride()} is invoked to notify the local node that some of
071 * its recent changes have been overridden; if all three attempts fail, then
072 * {@link #handleImpossibleMerge handleImpossibleMerge()} is invoked and we stay with whatever version we are on;
073 * this latter case can only occur when another node is running a different version of the software
074 * (with different validation criteria).
075 * </p>
076 *
077 * <p>
078 * For attempts #2 and #3, the "winner" is chosen as follows:
079 *  <ul>
080 *  <li>If the commits have different timestamps, then the later commit wins</li>
081 *  <li>If the two commits were simultaneous, then the commit with the lexicographically higher SHA-1 checksum wins</li>
082 *  <li>If the two commits have the same timestamp and SHA-1 checksum, then there won't be any conflicts :)</li>
083 *  </ul>
084 * </p>
085 *
086 * <p>
087 * Subclasses may {@linkplain #chooseConflictWinner override how the winner is chosen}.
088 * </p>
089 *
090 * <p>
091 * Because all nodes follow this same algorithm for merging, they will all end up with the same result
092 * (note however that the {@link PersistentObject} object graph must serialize in a deterministic way for this to be true).
093 * </p>
094 *
095 * <p>
096 * Note that merge conflicts can only happen when two nodes make changes at virtually the same time (or, when
097 * separated by a network partition): i.e., a change on one node occurs within the window of time after a change
098 * is made on another node but before the first node is notified and synchronizes. Using a simple
099 * real-time notification process, this window can be kept small, especially relative to the frequent-read,
100 * infrequent-write access pattern that the {@link PersistentObject} class is designed for. When a conflict override
101 * does occur, subclasses can override {@link #handleConflictOverride handleConflictOverride()} to take appropriate action.
102 * </p>
103 *
104 * <p>
105 * Instances of this class are thread safe.
106 * </p>
107 *
108 * @param <T> type of the root object
109 */
110public class Synchronizer<T> extends AbstractBean implements PersistentObjectListener<T> {

230    @Override
231    public void afterPropertiesSet() throws Exception {
232        super.afterPropertiesSet();
233
234        // Sanity check
235        if (this.persistentObject == null)
236            throw new Exception("no PersistentObject configured");
237        if (this.git == null)
238            throw new Exception("no GitRepository configured");
239        if (this.filename == null)
240            throw new Exception("no filename configured");
241        if (this.branch == null)
242            throw new Exception("no branch name configured");
243        if (!GitRepository.BRANCH_NAME_PATTERN.matcher(this.branch).matches())
244            throw new Exception("illegal branch name `" + this.branch + "'");
245        if (this.remotes == null)
246            throw new Exception("no remotes configured");
247        for (String remote : this.remotes) {
248            if (remote == null || !GitRepository.REMOTE_NAME_PATTERN.matcher(remote).matches())
249                throw new Exception("illegal remote name `" + remote + "'");
250        }
251
252        // Listen for POBJ changes
253        this.persistentObject.addListener(this);
254
255        // Do the first update
256        final PersistentObject<T>.Snapshot snapshot = this.persistentObject.getSharedRootSnapshot();
257        this.applyLocalUpdate(snapshot, "Initial update after startup (version " + snapshot.getVersion() + ")");
258    }
259
260    @Override
261    public void destroy() throws Exception {
262        this.persistentObject.removeListener(this);
263        this.git = null;
264        super.destroy();
265    }

267    @Override
268    public void handleEvent(PersistentObjectEvent<T> event) {
269        this.applyLocalUpdate(this.persistentObject.new Snapshot(event.getNewRoot(), event.getVersion()),
270          "Update after local change (version " + event.getVersion() + ")");
271    }

273    /**
274     * Given a (possibly) updated local root, {@linkplain #commit commit} it to our local branch,
275     * {@link #synchronize} with our peers if it represented a change, and apply the newly merged
276     * root (if any) back to the associated {@link PersistentObject}.
277     *
278     * <p>
279     * This method is invoked at {@linkplain #afterPropertiesSet startup} and after {@linkplain #handleEvent notification}
280     * of each local change.
281     * <p>
282     *
283     * <p>
284     * This method does nothing if the root is null.
285     * </p>
286     *
287     * @param snapshot snapshot containing new root (possibly null) and its version
288     * @param commitMessage message for the commit
289     */
290    protected synchronized void applyLocalUpdate(PersistentObject<T>.Snapshot snapshot, String commitMessage) {
291
292        // Get root; ignore if null
293        final T root = snapshot.getRoot();
294        if (root == null)
295            return;
296
297        // Commit new root
298        this.log.debug("committing new local root to branch `" + this.branch + "' (version " + snapshot.getVersion() + ")");
299        String commit;
300        try {
301            commit = this.commit(root, commitMessage);
302        } catch (Exception e) {
303            this.log.error("error committing local root to branch `" + this.branch + "'", e);
304            return;
305        }
306
307        // If nothing changed, nothing else to do
308        if (commit == null) {
309            this.log.debug("commit of version " + snapshot.getVersion() + " resulted in no change on branch `" + this.branch + "'");
310            return;
311        }
312        this.log.info("committed root version " + snapshot.getVersion() + " as " + commit + " on branch `" + this.branch + "'");
313
314        // Synchronize it; if no merge required or merge was trivial, we're done
315        if (!this.synchronize())
316            return;
317
318        // Read in the new, merged object graph
319        final ArrayList<T> rootList = new ArrayList<T>(1);
320        String mergeCommit;
321        try {
322            mergeCommit = this.git.access(this.branch, new GitRepository.Accessor() {
323                @Override
324                public void accessWorkingCopy(File dir) {
325                    rootList.add(Synchronizer.this.readXMLFile(dir, false));
326                }
327            });
328        } catch (PersistentObjectException e) {
329            this.log.error("error reading newly merged root; the local persistent object will not be updated", e);
330            return;
331        }
332
333        // Apply changes to local persistent object; but if the local version has changed, do nothing;
334        // we will do another merge when we get the next notification for the newer root.
335        try {
336            this.log.debug("applying merge commit " + mergeCommit + " to local persistent object");
337            final long version = this.persistentObject.setRoot(rootList.get(0), snapshot.getVersion());
338            this.log.info("successfully applied commit " + mergeCommit + " to local persistent object as version " + version);
339        } catch (PersistentObjectVersionException e) {
340            this.log.debug("merged root is out of date (version " + e.getExpectedVersion() + " < " + e.getActualVersion()
341              + "); will wait for next notification");
342        } catch (PersistentObjectException e) {
343            this.log.error("error applying newly merged root; the local persistent object will not be updated", e);
344        }
345    }

366    /**
367     * Commit the given root to our {@code git(1)} repository.
368     *
369     * <p>
370     * If the local branch does not already exist, it will be created.
371     * </p>
372     *
373     * @param root persistent object root
374     * @param commitMessage message for the commit
375     * @return new commit ID if commit occurred, or null if {@code root} was identical to the already-committed head of our branch
376     * @throws IllegalArgumentException if {@code root} is null
377     * @throws IllegalArgumentException if {@code commitMessage} is null
378     */
379    protected synchronized String commit(final T root, String commitMessage) {
380
381        // Sanity check
382        if (root == null)
383            throw new IllegalArgumentException("null root");
384        if (commitMessage == null)
385            throw new IllegalArgumentException("null commitMessage");
386
387        // Ensure branch exists
388        this.git.ensureBranch(this.branch, "Empty commit as the basis for branch `" + this.branch + "'");
389
390        // Get previous commit ID
391        final String previousCommit = this.git.followReference("refs/heads/" + this.branch);
392
393        // Commit it
394        this.log.debug("committing root to local branch `" + this.branch + "'");
395        final String newCommit = this.git.commit(this.branch, new GitRepository.Accessor() {
396            @Override
397            public void accessWorkingCopy(File dir) {
398                PersistentObject.write(root, Synchronizer.this.persistentObject.getDelegate(),
399                  Synchronizer.this.getXMLFile(dir));
400            }
401        }, commitMessage);
402
403        // Report results
404        return !newCommit.equals(previousCommit) ? newCommit : null;
405    }

407    /**
408     * Synchronize what we have previously committed with our peers' latest information.
409     *
410     * <p>
411     * This method fetches the configured remotes and then merges them into our local branch,
412     * ensuring that the merge always yeilds a root that validates.
413     * </p>
414     *
415     * <p>
416     * If the local branch does not already exist, it will be created.
417     * </p>
418     *
419     * <p>
420     * This method does not affect the {@link PersistentObject} associated with this instance;
421     * the caller is responsible for doing that if necessary (i.e., true is returned).
422     * </p>
423     *
424     * @return true if a non-trivial merge ocurred, false if we were already up-to-date
425     */
426    public synchronized boolean synchronize() {
427
428        // Fetch the latest from our remotes
429        this.log.debug("beginning synchronization with remotes: " + this.remotes);
430        this.git.fetch(this.remotes);
431
432        // Ensure local branch exists
433        this.git.ensureBranch(this.branch, "Empty commit as the basis for branch `" + this.branch + "'");
434
435        // Merge our branch with each remote's branch
436        boolean merged = false;
437    remoteLoop:
438        for (String remote : this.remotes) {
439
440            // Get remote commit ID and date
441            final String remoteRef = "refs/remotes/" + remote + "/" + this.branch;
442            if (!this.git.getRepoFile(remoteRef).exists()) {
443                this.log.debug("remote `" + remote + "' branch `" + this.branch + "' does not exist yet, skipping this remote");
444                continue;
445            }
446            final String remoteCommit = this.git.followReference(remoteRef);
447
448            // Get local commit ID and date
449            final String localRef = "refs/heads/" + this.branch;
450            final String localCommit = this.git.followReference(localRef);
451
452            // Compare trees for equality
453            if (this.git.equalTrees(localCommit, remoteCommit)) {
454                this.log.debug("remote `" + remote + "' commit " + remoteCommit + " matches our branch `"
455                  + this.branch + "' commit " + localCommit + ", skipping this remote");
456                continue;
457            }
458
459            // Logit
460            this.log.info("merging remote `" + remote + "' commit " + remoteCommit + " with local commit " + localCommit);
461
462            // Determine who is the winner in case of conflicts
463            boolean iWin = localCommit != null && this.chooseConflictWinner(localCommit, remoteCommit);
464
465            // Build merge strategy list
466            final MergeStrategy[] strategyList = new MergeStrategy[] {
467              MergeStrategy.RECURSIVE_PATIENCE,
468              iWin ? MergeStrategy.RECURSIVE_PATIENCE_OURS : MergeStrategy.RECURSIVE_PATIENCE_THEIRS,
469              iWin ? MergeStrategy.RECURSIVE_OURS : MergeStrategy.RECURSIVE_THEIRS
470            };
471
472            // Attempt merges using successively less "mergey" strategies
473            for (MergeStrategy strategy : strategyList) {
474
475                // Attempt merge
476                this.log.debug("attempting merge of remote `" + remote + "' commit " + remoteCommit
477                  + " with local commit " + localCommit + " using strategy " + strategy);
478                String commit;
479                try {
480                    commit = this.git.merge(this.branch, remoteCommit, strategy, new GitRepository.Accessor() {
481                        @Override
482                        public void accessWorkingCopy(File dir) {
483                            Synchronizer.this.log.debug("validating merged file `" + Synchronizer.this.getXMLFile(dir) + "'");
484                            Synchronizer.this.readXMLFile(dir, true);
485                        }
486                    }, "Merged remote `" + remote + "' commit " + remoteCommit + " using strategy " + strategy);
487                } catch (GitMergeConflictException e) {
488                    this.log.debug("merge using strategy " + strategy + " failed with conflict(s)");
489                    continue;
490                } catch (PersistentObjectException e) {
491                    this.log.debug("merge using strategy " + strategy + " did not validate: " + e);
492                    continue;
493                }
494
495                // We have a winner
496                if (commit.equals(localCommit)) {
497                    this.log.debug("merge of remote `" + remote + "' commit " + remoteCommit + " with local commit "
498                      + localCommit + " using strategy " + strategy + " resulted in no change to our version");
499                } else {
500                    this.log.debug("successfully merged remote `" + remote + "' commit " + remoteCommit + " with local commit "
501                      + localCommit + " using strategy " + strategy + " resulting in commit " + commit);
502                    merged = true;
503                }
504
505                // Log a warning if some of our local changes got overridden by remote
506                switch (strategy) {
507                case RECURSIVE_PATIENCE_THEIRS:
508                case RECURSIVE_THEIRS:
509                    this.handleConflictOverride(remote, localCommit, remoteCommit, strategy);
510                    break;
511                default:
512                    break;
513                }
514
515                // Done with this remote
516                continue remoteLoop;
517            }
518
519            // Nothing worked - so we must be running different versions of the application
520            this.handleImpossibleMerge(remote, localCommit, remoteCommit);
521        }
522
523        // Done
524        this.log.debug("completed synchronization with remotes " + this.remotes
525          + " (" + (merged ? "a" : "no") + " merge was required)");
526        return merged;
527    }

529    /**
530     * Choose which commit should "win" in case there is a conflict between two commits.
531     *
532     * <p>
533     * It is imperative that this method <b>only</b> depend on information associated with the given commits.
534     * In particular, it must not depend on which node it is running on. This ensures that all nodes come
535     * to the same conclusion and therefore produce the same outcome when merging.
536     * </p>
537     *
538     * <p>
539     * The implementation in {@link Synchronizer} chooses the commit with the later timestamp, or if both commits
540     * have the same timestamp, then the commit with the lexicographically higher SHA-1 checksum.
541     * </p>
542     *
543     * @param commit1 first commit ID
544     * @param commit2 second commit ID
545     * @return true if {@code commit1} should win, false if {@code commit2} should win
546     */
547    protected boolean chooseConflictWinner(String commit1, String commit2) {
548        final Date date1 = this.git.getAuthorDate(commit1);
549        final Date date2 = this.git.getAuthorDate(commit2);
550        return date1.compareTo(date2) > 0 || (date1.compareTo(date2) == 0 && commit1.compareTo(commit2) > 0);
551    }

553    /**
554     * Handle a situation where some of our local changes were overridden due to merge conflicts, but
555     * in the end we were able to produce a valid root.
556     *
557     * <p>
558     * The implementation in {@link Synchronizer} just logs a warning message.
559     * </p>
560     *
561     * @param remote name of the remote that overrode us
562     * @param localCommit our commit in the merge
563     * @param remoteCommit remote's commit in the merge
564     * @param strategy the strategy that was used to resolve the conflict
565     */
566    protected void handleConflictOverride(String remote, String localCommit, String remoteCommit, MergeStrategy strategy) {
567        this.log.warn("some changes in local commit " + localCommit + " have been overridden by changes from remote `"
568          + remote + "' commit " + remoteCommit + " using merge strategy " + strategy);
569    }

571    /**
572     * Handle a situation where all of the merge strategies failed to produce a valid root object.
573     * This can only happen when a remote node has different validation criteria than we do, for example,
574     * it is running a different version of the application.
575     *
576     * <p>
577     * The implementation in {@link Synchronizer} just logs a warning message.
578     * </p>
579     *
580     * @param remote name of the remote that we are conflicting with
581     * @param localCommit our commit in the merge
582     * @param remoteCommit remote's commit in the merge
583     */
584    protected void handleImpossibleMerge(String remote, String localCommit, String remoteCommit) {
585        this.log.error("unable to merge local commit " + localCommit + " with remote `" + remote + "' commit "
586          + remoteCommit + "; are we running different application versions?");
587    }
