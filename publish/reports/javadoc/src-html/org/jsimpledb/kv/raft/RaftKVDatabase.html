089/**
090 * A distributed {@link org.jsimpledb.kv.KVDatabase} based on the Raft consensus algorithm.
091 *
092 * <p>
093 * Raft defines a distributed consensus algorithm for maintaining a shared state machine.
094 * Each Raft node maintains a complete copy of the state machine. Cluster nodes elect a
095 * leader who collects and distributes updates and provides for consistent reads.
096 * As long as as a node is part of a majority, the state machine is fully operational.
097 *
098 * <p>
099 * {@link RaftKVDatabase} turns this into a transactional key/value database with linearizable ACID semantics as follows:
100 *  <ul>
101 *  <li>The Raft state machine is the key/value store data.</li>
102 *  <li>Unapplied log entries are stored on disk as serialized mutations, and also cached in memory.</li>
103 *  <li>On leaders only, committed log entries are not applied immediately; instead they kept around for time at least
104 *      <i>T<sub>max</sub></i> since creation, where <i>T<sub>max</sub></i> is the maximum supported transaction duration.
105 *      This caching is subject to <i>M<sub>max</sub></i>, which limits the total memory used; if reached, these log entries
106 *      are applied early.</li>
107 *  <li>Concurrent transactions are supported through a simple optimistic locking MVCC scheme (same as used by
108 *      {@link org.jsimpledb.kv.mvcc.SnapshotKVDatabase}):
109 *      <ul>
110 *      <li>Transactions execute locally until commit time, using a {@link MutableView} to collect mutations.
111 *          The {@link MutableView} is based on the local node's most recent log entry (whether committed or not);
112 *          this is called the <i>base term and index</i> for the transaction.</li>
113 *      <li>On commit, the transaction's {@link Reads}, {@link Writes}, base index and term, and any config change are
114 *          {@linkplain CommitRequest sent} to the leader.</li>
115 *      <li>The leader confirms that the log entry corresponding to the transaction's base index is either not yet applied,
116 *          or was its most recently applied log entry. If this is not the case, then the transaction's base log entry
117 *          is too old (older than <i>T<sub>max</sub></i>, or was applied and discarded early due to memory pressure),
118 *          and so the transaction is rejected with a {@link RetryTransactionException}.
119 *      <li>The leader verifies that the the log entry term matches the transaction's base term; if not, the base log entry
120 *          has been overwritten, and the transaction is rejected with a {@link RetryTransactionException}.
121 *      <li>The leader confirms that the {@link Writes} associated with log entries after the transaction's base log entry
122 *          do not create {@linkplain Reads#isConflict conflicts} when compared against the transaction's {@link Reads}.
123 *          If so, the transaction is rejected with a {@link RetryTransactionException}.</li>
124 *      <li>The leader adds a new log entry consisting of the transaction's {@link Writes} (and any config change) to its log.
125 *          The associated term and index become the transaction's <i>commit term and index</i>; the leader then
126 *          {@linkplain CommitResponse replies} to the follower with this information.</li>
127 *      <li>If/when the follower sees a <i>committed</i> (in the Raft sense) log entry appear in its log matching the
128 *          transaction's commit term and index, then the transaction is complete.</li>
129 *      <li>As an optimization, when the leader sends a log entry to the same follower who committed the corresponding
130 *          transaction in the first place, only the transaction ID is sent, because the follower already has the data.</li>
131 *      </ul>
132 *  </li>
133 *  <li>For transactions occurring on a leader, the logic is similar except of course no network communication occurs.</li>
134 *  <li>For read-only transactions, the leader does not create a new log entry; instead, the transaction's commit
135 *      term and index are set to the base term and index, and the leader also calculates its current "leader lease timeout",
136 *      which is the earliest time at which it is possible for another leader to be elected.
137 *      This is calculated as the time in the past at which the leader sent {@link AppendRequest}'s to a majority of followers
138 *      who have since responded, plus the {@linkplain #setMinElectionTimeout minimum election timeout}, minus a small adjustment
139 *      for possible clock drift (this assumes all nodes have the same minimum election timeout configured). If the current
140 *      time is prior to the leader lease timeout, the transaction may be committed immediately; otherwise, the current time
141 *      is returned to the follower as minimum required leader lease timeout before the transaction may be committed.</li>
142 *  <li>Every {@link AppendRequest} includes the leader's current timestamp and leader lease timeout, so followers can commit
143 *      any waiting read-only transactions. Leaders keep track of which followers are waiting on which leader lease
144 *      timeout values, and when the leader lease timeout advances to allow a follower to commit a transaction, the follower
145 *      is immediately notified.</li>
146 *  <li>A weaker consistency guarantee for read-only transactions (stale reads) is also possible; see
147 *      {@link RaftKVTransaction#setReadOnlySnapshot RaftKVTransaction.setReadOnlySnapshot()}. Typically these
148 *      transactions will generate no network traffic.</li>
149 *  </ul>
150 *
151 * <p><b>Limitations</b></p>
152 *
153 * <ul>
154 *  <li>A transaction's mutations must fit in memory.</li>
155 *  <li>An {@link AtomicKVStore} is required to store local persistent state;
156 *      if none is configured, a {@link LevelDBKVStore} is used.</li>
157 *  <li>All nodes must be configured with the same {@linkplain #setMinElectionTimeout minimum election timeout}.
158 *      This guarantees that the leader's lease timeout calculation is valid.</li>
159 *  <li>Due to the optimistic locking approach used, this implementation will perform poorly when there is a high
160 *      rate of conflicting transactions; the result will be many transaction retries.</li>
161 *  <li>Performance will suffer when the amount of data associated with a typical transaction cannot be delivered
162 *      quickly and reliably over the network.</li>
163 * </ul>
164 *
165 * <p>
166 * In general, the algorithm should function correctly under all non-Byzantine conditions. The level of difficultly
167 * the system is experiencing, due to contention, network errors, etc., can be measured in terms of:
168 * <ul>
169 *  <li>The average amount of time it takes to commit a transaction</li>
170 *  <li>The frequency of {@link RetryTransactionException}'s</li>
171 * </ul>
172 *
173 * <p><b>Cluster Configuration</b></p>
174 *
175 * <p>
176 * Instances support dynamic cluster configuration changes at runtime.
177 *
178 * <p>
179 * Initially, all nodes are in an <i>unconfigured</i> state, where nothing has been added to the Raft log yet and no
180 * cluster is defined. Unconfigured nodes are passive: they stay in follower mode (i.e., they will not start elections),
181 * and they disallow local transactions that make any changes other than as described below to initialize a new cluster.
182 *
183 * <p>
184 * An unconfigured node becomes configured when either:
185 * <ol>
186 *  <li>{@link RaftKVTransaction#configChange RaftKVTransaction.configChange()} is invoked and committed within
187 *      a local transaction, which creates a new single node cluster and commits the first log entry; or</li>
188 *  <li>An {@link AppendRequest} is received from a leader of some existing cluster, in which case the node
189 *      records the cluster ID (see below) and applies the received cluster configuration</li>
190 * </ol>
191 *
192 * <p>
193 * A node is configured if and only if it has recorded one or more log entries. The very first log entry
194 * always contains the initial cluster configuration (containing only the node that created it), so any node that has a
195 * non-empty log is configured.
196 *
197 * <p>
198 * Newly created clusters are assigned a random 32-bit cluster ID (option #1 above). This ID is included in all messages sent
199 * over the network, and adopted by unconfigured nodes that join the cluster (via option #2 above). Nodes discard incoming
200 * messages containing a cluster ID different from one they have seen previously. This prevents data corruption that can occur
201 * if nodes from two different clusters are inadvertently "mixed" together.
202 *
203 * <p>
204 * Once a node joins a cluster with a specific cluster ID, it cannot be reassigned to a different cluster without first
205 * returning it to the unconfigured state; to do that, it must be shut it down and its persistent state deleted.
206 *
207 * <p><b>Configuration Changes</b></p>
208 *
209 * <p>
210 * Once a node configured, a separate issue is whether the node is <i>included</i> in its own configuration, i.e., whether
211 * the node is a member of its cluster. A node that is not a member of its cluster does not count its own vote to determine
212 * committed log entries (if a leader), and does not start elections (if a follower). However, it will accept and respond
213 * to incoming {@link AppendRequest}s and {@link RequestVote}s.
214 *
215 * <p>
216 * In addition, leaders follow these rules:
217 * <ul>
218 *  <li>If a leader is removed from a cluster, it remains the leader until the configuration change that
219 *      removed it is committed (not counting its own vote), and then steps down (reverts to follower).</li>
220 *  <li>If a follower is added to a cluster, the leader immediately starts sending that follower {@link AppendRequest}s.</li>
221 *  <li>If a follower is removed from a cluster, the leader continues to send that follower {@link AppendRequest}s
222 *      until the follower acknowledges receipt of the log entry containing the configuration change.</li>
223 *  <li>Configuration changes that remove the last node in a cluster are disallowed.</li>
224 *  <li>Only one configuration change may take place at a time.</li>
225 * </ul>
226 *
227 * @see <a href="https://raftconsensus.github.io/">The Raft Consensus Algorithm</a>
228 */
229public class RaftKVDatabase implements KVDatabase {

537     * Start this instance.
538     *
539     * <p>
540     * Does nothing if already started or in the process of shutting down.
541     * </p>

543     * @throws IllegalStateException if this instance is not properly configured

547    @PostConstruct
548    public synchronized void start() throws IOException {
549
550        // Sanity check
551        assert this.checkState();
552        if (this.role != null)
553            return;
554        Preconditions.checkState(!this.shuttingDown, "shutdown in progress");
555        Preconditions.checkState(this.logDir != null, "no log directory configured");
556        Preconditions.checkState(this.network != null, "no network configured");
557        Preconditions.checkState(this.kv == null, "key/value store exists");
558        Preconditions.checkState(this.minElectionTimeout <= this.maxElectionTimeout, "minElectionTimeout > maxElectionTimeout");
559        Preconditions.checkState(this.heartbeatTimeout < this.minElectionTimeout, "heartbeatTimeout >= minElectionTimeout");
560        Preconditions.checkState(this.identity != null, "no identity configured");
561
562        // Log
563        this.info("starting " + this + " in directory " + this.logDir);
564
565        // Start up local database
566        boolean success = false;
567        try {
568
569            // Create/verify log directory
570            if (!this.logDir.exists() && !this.logDir.isDirectory())
571                throw new IOException("failed to create directory `" + this.logDir + "'");
572            if (!this.logDir.isDirectory())
573                throw new IOException("file `" + this.logDir + "' is not a directory");
574
575            // By default use an atomic key/value store based on LevelDB if not explicitly specified
576            if (this.kvstore != null)
577                this.kv = this.kvstore;
578            else {
579                final File leveldbDir = new File(this.logDir, "levedb" + KVSTORE_FILE_SUFFIX);
580                if (!leveldbDir.exists() && !leveldbDir.mkdirs())
581                    throw new IOException("failed to create directory `" + leveldbDir + "'");
582                if (!leveldbDir.isDirectory())
583                    throw new IOException("file `" + leveldbDir + "' is not a directory");
584                this.kv = new LevelDBKVStore(new Iq80DBFactory().open(leveldbDir, new Options().createIfMissing(true)));
585            }
586
587            // Open directory containing log entry files so we have a way to fsync() it
588            assert this.logDirChannel == null;
589            this.logDirChannel = FileChannel.open(this.logDir.toPath(), StandardOpenOption.READ);
590
591            // Create randomizer
592            assert this.random == null;
593            this.random = new SecureRandom();
594
595            // Start up executor thread
596            assert this.executor == null;
597            this.executor = Executors.newSingleThreadScheduledExecutor(new ThreadFactory() {
598                @Override
599                public Thread newThread(Runnable action) {
600                    return new ServiceThread(action);
601                }
602            });
603
604            // Start network
605            this.network.start(new Network.Handler() {
606                @Override
607                public void handle(String sender, ByteBuffer buf) {
608                    RaftKVDatabase.this.handle(sender, buf);
609                }
610
611                @Override
612                public void outputQueueEmpty(String address) {
613                    RaftKVDatabase.this.outputQueueEmpty(address);
614                }
615            });
616
617            // Reload persistent raft info
618            this.clusterId = (int)this.decodeLong(CLUSTER_ID_KEY, 0);
619            this.currentTerm = this.decodeLong(CURRENT_TERM_KEY, 0);
620            final String votedFor = this.decodeString(VOTED_FOR_KEY, null);
621            this.lastAppliedTerm = this.decodeLong(LAST_APPLIED_TERM_KEY, 0);
622            this.lastAppliedIndex = this.decodeLong(LAST_APPLIED_INDEX_KEY, 0);
623            this.lastAppliedConfig = this.decodeConfig(LAST_APPLIED_CONFIG_KEY);
624            this.currentConfig = this.buildCurrentConfig();
625
626            // If we crashed part way through a snapshot install, recover by resetting state machine
627            if (this.lastAppliedTerm < 0 || this.lastAppliedIndex < 0) {
628                this.info("detected partially applied snapshot, resetting state machine");
629                if (!this.resetStateMachine(true))
630                    throw new IOException("error resetting state machine");
631            }
632
633            // Initialize commit index
634            this.commitIndex = this.lastAppliedIndex;
635
636            // Reload outstanding log entries from disk
637            this.loadLog();
638
639            // Show recovered state
640            this.info("recovered Raft state:"
641              + "\n  clusterId=" + (this.clusterId != 0 ? String.format("0x%08x", this.clusterId) : "none")
642              + "\n  currentTerm=" + this.currentTerm
643              + "\n  lastApplied=" + this.lastAppliedIndex + "t" + this.lastAppliedTerm
644              + "\n  lastAppliedConfig=" + this.lastAppliedConfig
645              + "\n  currentConfig=" + this.currentConfig
646              + "\n  votedFor=" + (votedFor != null ? "\"" + votedFor + "\"" : "nobody")
647              + "\n  log=" + this.raftLog);
648
649            // Validate recovered state
650            if (this.isConfigured()) {
651                Preconditions.checkArgument(this.clusterId != 0);
652                Preconditions.checkArgument(this.currentTerm > 0);
653                Preconditions.checkArgument(this.getLastLogTerm() > 0);
654                Preconditions.checkArgument(this.getLastLogIndex() > 0);
655                Preconditions.checkArgument(!this.currentConfig.isEmpty());
656            } else {
657                Preconditions.checkArgument(this.lastAppliedTerm == 0);
658                Preconditions.checkArgument(this.lastAppliedIndex == 0);
659                Preconditions.checkArgument(this.currentTerm == 0);
660                Preconditions.checkArgument(this.getLastLogTerm() == 0);
661                Preconditions.checkArgument(this.getLastLogIndex() == 0);
662                Preconditions.checkArgument(this.currentConfig.isEmpty());
663                Preconditions.checkArgument(this.raftLog.isEmpty());
664            }
665
666            // Start as follower (with unknown leader)
667            this.changeRole(new FollowerRole(this, null, null, votedFor));
668
669            // Done
670            success = true;
671        } finally {
672            if (!success)
673                this.cleanup();
674        }
675
676        // Sanity check
677        assert this.checkState();
678    }

680    /**
681     * Stop this instance.
682     *
683     * <p>
684     * Does nothing if not {@linkplain #start started} or in the process of shutting down.
685     * </p>
686     */
687    @PreDestroy
688    public void stop() {
689
690        // Set flag to prevent new transactions
691        synchronized (this) {
692
693            // Sanity check
694            assert this.checkState();
695            if (this.role == null || this.shuttingDown)
696                return;
697
698            // Set shutting down flag
699            this.info("starting shutdown of " + this);
700            this.shuttingDown = true;
701
702            // Fail all remaining open transactions
703            for (RaftKVTransaction tx : new ArrayList<RaftKVTransaction>(this.openTransactions.values())) {
704                switch (tx.getState()) {
705                case EXECUTING:
706                    tx.rollback();
707                    break;
708                case COMMIT_READY:
709                case COMMIT_WAITING:
710                    this.fail(tx, new KVTransactionException(tx, "database shutdown"));
711                    break;
712                case COMPLETED:
713                    break;
714                default:
715                    assert false;
716                    break;
717                }
718            }
719
720            // Sleep while we wait for transactions to clean themselves up
721            boolean done = false;
722            try {
723                done = TimedWait.wait(this, 5000, new org.dellroad.stuff.java.Predicate() {
724                    @Override
725                    public boolean test() {
726                        return RaftKVDatabase.this.openTransactions.isEmpty();
727                    }
728                });
729            } catch (InterruptedException e) {
730                Thread.currentThread().interrupt();
731            }
732            if (!done)
733                this.warn("open transactions not cleaned up during shutdown");
734        }
735
736        // Shut down the executor and wait for pending tasks to finish
737        this.executor.shutdownNow();
738        try {
739            this.executor.awaitTermination(1000, TimeUnit.MILLISECONDS);
740        } catch (InterruptedException e) {
741            Thread.currentThread().interrupt();
742        }
743
744        // Final cleanup
745        synchronized (this) {
746            this.executor = null;
747            this.cleanup();
748        }
749
750        // Done
751        this.info("completed shutdown of " + this);
752    }

754    private void cleanup() {
755        assert Thread.holdsLock(this);
756        assert this.openTransactions.isEmpty();
757        if (this.role != null) {
758            this.role.shutdown();
759            this.role = null;
760        }
761        if (this.executor != null) {
762            this.executor.shutdownNow();
763            try {
764                this.executor.awaitTermination(1000, TimeUnit.MILLISECONDS);
765            } catch (InterruptedException e) {
766                Thread.currentThread().interrupt();
767            }
768            this.executor = null;
769        }
770        if (this.kv != null) {
771            if (this.kvstore == null) {
772                final LevelDBKVStore ldb = (LevelDBKVStore)this.kv;
773                Util.closeIfPossible(ldb);
774                Util.closeIfPossible(ldb.getDB());
775            }
776            this.kv = null;
777        }
778        Util.closeIfPossible(this.logDirChannel);
779        this.logDirChannel = null;
780        this.raftLog.clear();
781        this.random = null;
782        this.network.stop();
783        this.currentTerm = 0;
784        this.commitIndex = 0;
785        this.clusterId = 0;
786        this.lastAppliedTerm = 0;
787        this.lastAppliedIndex = 0;
788        this.lastAppliedConfig = null;
789        this.currentConfig = null;
790        this.transmitting.clear();
791        this.pendingService.clear();
792        this.shuttingDown = false;
793    }

795    /**
796     * Initialize our in-memory state from the persistent state reloaded from disk.
797     * This is invoked on initial startup.
798     */
799    private void loadLog() throws IOException {
800
801        // Sanity check
802        assert Thread.holdsLock(this);
803
804        // Scan for log entry files
805        this.raftLog.clear();
806        for (File file : this.logDir.listFiles()) {
807
808            // Is this a log entry file?
809            if (LogEntry.LOG_FILE_PATTERN.matcher(file.getName()).matches()) {
810                this.info("recovering log file " + file.getName());
811                this.raftLog.add(LogEntry.fromFile(file));
812                continue;
813            }
814
815            // Is this a leftover temporary file?
816            if (TEMP_FILE_PATTERN.matcher(file.getName()).matches()) {
817                this.info("deleting leftover temporary file " + file.getName());
818                if (!file.delete())
819                    this.error("failed to delete leftover temporary file " + file);
820                continue;
821            }
822
823            // Is this a KV store directory (expected)?
824            if (KVSTORE_FILE_PATTERN.matcher(file.getName()).matches())
825                continue;
826
827            // Unknown
828            this.warn("ignoring unrecognized file " + file.getName() + " in my log directory");
829        }
830
831        // Verify we have a contiguous range of log entries starting from the snapshot index; discard bogus log files
832        Collections.sort(this.raftLog, LogEntry.SORT_BY_INDEX);
833        long lastTermSeen = this.lastAppliedTerm;
834        long expectedIndex = this.lastAppliedIndex + 1;
835        for (Iterator<LogEntry> i = this.raftLog.iterator(); i.hasNext(); ) {
836            final LogEntry logEntry = i.next();
837            String error = null;
838            if (logEntry.getTerm() < lastTermSeen)
839                error = "term " + logEntry.getTerm() + " < last applied term " + lastTermSeen;
840            else if (logEntry.getIndex() < this.lastAppliedIndex)
841                error = "index " + logEntry.getIndex() + " < last applied index " + this.lastAppliedIndex;
842            else if (logEntry.getIndex() != expectedIndex)
843                error = "index " + logEntry.getIndex() + " != expected index " + expectedIndex;
844            if (error != null) {
845                this.warn("deleting bogus log file " + logEntry.getFile().getName() + ": " + error);
846                if (!logEntry.getFile().delete())
847                    this.error("failed to delete bogus log file " + logEntry.getFile());
848                i.remove();
849            } else {
850                expectedIndex++;
851                lastTermSeen = logEntry.getTerm();
852            }
853        }
854        this.info("recovered " + this.raftLog.size() + " Raft log entries: " + this.raftLog);
855
856        // Rebuild current configuration
857        this.currentConfig = this.buildCurrentConfig();
858    }

860    private Map<String, String> buildCurrentConfig() {
861
862        // Start with last applied config
863        final HashMap<String, String> config = new HashMap<>(this.lastAppliedConfig);
864
865        // Apply any changes found in uncommitted log entries
866        for (LogEntry logEntry : this.raftLog)
867            logEntry.applyConfigChange(config);
868
869        // Done
870        return config;

940     *
941     * @throws IllegalStateException if this instance is not {@linkplain #start started} or in the process of shutting down
942     */
943    @Override
944    public synchronized RaftKVTransaction createTransaction() {
945
946        // Sanity check
947        assert this.checkState();
948        Preconditions.checkState(this.role != null, "not started");
949        Preconditions.checkState(!this.shuttingDown, "shutting down");
950
951        // Base transaction on the most recent log entry. This is itself a form of optimistic locking: we assume that the
952        // most recent log entry has a high probability of being committed (in the Raft sense), which is of course required
953        // in order to commit any transaction based on it.
954        final MostRecentView view = new MostRecentView();
955
956        // Create transaction
957        final RaftKVTransaction tx = new RaftKVTransaction(this,
958          this.getLastLogTerm(), this.getLastLogIndex(), view.getSnapshot(), view.getView());
959        if (this.log.isDebugEnabled())
960            this.debug("created new transaction " + tx);
961        this.openTransactions.put(tx.getTxId(), tx);
962
963        // Set default commit timeout
964        tx.setTimeout(this.commitTimeout);
965
966        // Done
967        return tx;
968    }

970    /**
971     * Commit a transaction.
972     */
973    void commit(final RaftKVTransaction tx) {
974        try {
975
976            // Mark transaction as "commit ready" - service thread will do the rest
977            synchronized (this) {
978
979                // Sanity check
980                assert this.checkState();
981                assert this.role != null;
982
983                // Check tx state
984                switch (tx.getState()) {
985                case EXECUTING:
986
987                    // Transition to COMMIT_READY state
988                    if (this.log.isDebugEnabled())
989                        this.debug("committing transaction " + tx);
990                    tx.closeSnapshot();
991                    tx.setState(TxState.COMMIT_READY);
992                    this.requestService(this.role.new CheckReadyTransactionService(tx));
993
994                    // Setup commit timer
995                    final int timeout = tx.getTimeout();
996                    if (timeout != 0) {
997                        final Timer commitTimer = new Timer("commit timer for " + tx,
998                          new Service(null, "commit timeout for tx#" + tx.getTxId()) {
999                            @Override
1000                            public void run() {
1001                                switch (tx.getState()) {
1002                                case COMMIT_READY:
1003                                case COMMIT_WAITING:
1004                                    RaftKVDatabase.this.fail(tx, new RetryTransactionException(tx,
1005                                      "transaction failed to complete within " + timeout + "ms (in state " + tx.getState() + ")"));
1006                                    break;
1007                                default:
1008                                    break;
1009                                }
1010                            }
1011                        });
1012                        commitTimer.timeoutAfter(tx.getTimeout());
1013                        tx.setCommitTimer(commitTimer);
1014                    }
1015                    break;
1016                case CLOSED:                                        // this transaction has already been committed or rolled back
1017                    throw new StaleTransactionException(tx);
1018                default:                                            // another thread is already doing the commit
1019                    this.warn("simultaneous commit()'s requested for " + tx + " by two different threads");
1020                    break;
1021                }
1022            }
1023
1024            // Wait for completion
1025            try {
1026                tx.getCommitFuture().get();
1027            } catch (InterruptedException e) {
1028                throw new RetryTransactionException(tx, "thread interrupted while waiting for commit", e);
1029            } catch (ExecutionException e) {
1030                final Throwable cause = e.getCause();
1031                Util.prependCurrentStackTrace(cause, "Asynchronous Commit");
1032                if (cause instanceof Error)
1033                    throw (Error)cause;
1034                if (cause instanceof RuntimeException)
1035                    throw (RuntimeException)cause;
1036                throw new KVTransactionException(tx, "commit failed", cause);           // should never get here
1037            }
1038        } finally {
1039            this.cleanupTransaction(tx);
1040        }
1041    }

1044     * Rollback a transaction.
1045     */
1046    synchronized void rollback(RaftKVTransaction tx) {
1047
1048        // Sanity check
1049        assert this.checkState();
1050        assert this.role != null;
1051
1052        // Check tx state
1053        switch (tx.getState()) {

1060            break;

1063            break;

1067    private synchronized void cleanupTransaction(RaftKVTransaction tx) {
1068
1069        // Debug
1070        if (this.log.isTraceEnabled())
1071            this.trace("cleaning up transaction " + tx);
1072
1073        // Do any per-role cleanups
1074        if (this.role != null)
1075            this.role.cleanupForTransaction(tx);
1076
1077        // Close transaction's snapshot
1078        tx.closeSnapshot();
1079
1080        // Cancel commit timer
1081        final Timer commitTimer = tx.getCommitTimer();
1082        if (commitTimer != null)
1083            commitTimer.cancel();
1084
1085        // Remove from open transactions set
1086        this.openTransactions.remove(tx.getTxId());
1087
1088        // Transition to CLOSED
1089        tx.setState(TxState.CLOSED);
1090
1091        // Notify waiting thread if doing shutdown
1092        if (this.shuttingDown)
1093            this.notify();
1094    }

1132    /**
1133     * A view of the database based on the most recent log entry, if any, otherwise directly on the committed key/value store.
1134     * Caller is responsible for eventually closing the snapshot.
1135     */
1136    private class MostRecentView {
1137
1138        private final CloseableKVStore snapshot;
1139        private final MutableView view;
1140
1141        public MostRecentView() {
1142            assert Thread.holdsLock(RaftKVDatabase.this);
1143
1144            // Grab a snapshot of the key/value store
1145            this.snapshot = RaftKVDatabase.this.kv.snapshot();
1146
1147            // Create a view of just the state machine keys and values and successively layer unapplied log entries
1148            KVStore kview = PrefixKVStore.create(snapshot, STATE_MACHINE_PREFIX);
1149            for (LogEntry logEntry : RaftKVDatabase.this.raftLog) {
1150                final Writes writes = logEntry.getWrites();
1151                if (!writes.isEmpty())
1152                    kview = new MutableView(kview, null, logEntry.getWrites());
1153            }
1154
1155            // Finalize
1156            this.view = new MutableView(kview);
1157        }
1158
1159        public CloseableKVStore getSnapshot() {
1160            return this.snapshot;
1161        }
1162
1163        public MutableView getView() {
1164            return this.view;
1165        }
1166    }

1170    /**
1171     * Request service to be invoked after the current service (if any) completes.
1172     *
1173     * <p>
1174     * If {@code service} has an associated {@link Role}, and that {@link Role} is no longer active
1175     * when the service is handled, nothing will be done.
1176     *
1177     * @param service the service to perform
1178     */
1179    private void requestService(Service service) {
1180        assert Thread.holdsLock(this);
1181        if (!this.pendingService.add(service))
1182            return;
1183        if (this.performingService)
1184            return;
1185        try {
1186            this.executor.submit(new ErrorLoggingRunnable() {
1187                @Override
1188                protected void doRun() {
1189                    RaftKVDatabase.this.handlePendingService();
1190                }
1191            });
1192        } catch (RejectedExecutionException e) {
1193            if (!this.shuttingDown)
1194                this.warn("executor task rejected, skipping", e);
1195        }
1196    }

1198    // Performs pending service requests (do not invoke directly)
1199    private synchronized void handlePendingService() {
1200
1201        // Sanity check
1202        assert this.checkState();
1203        assert Thread.currentThread() instanceof ServiceThread;
1204        if (this.role == null)
1205            return;
1206
1207        // While there is work to do, do it
1208        this.performingService = true;
1209        try {
1210            while (!this.pendingService.isEmpty()) {
1211                final Iterator<Service> i = this.pendingService.iterator();
1212                final Service service = i.next();
1213                i.remove();
1214                assert service.getRole() == null || service.getRole() == this.role;
1215                if (this.log.isTraceEnabled())
1216                    this.trace("SERVICE [" + service + "] in " + this.role);
1217                service.run();
1218            }
1219        } finally {
1220            this.performingService = false;
1221        }
1222    }

1253    /**
1254     * One shot timer that {@linkplain #requestService requests} a {@link Service} on expiration.
1255     *
1256     * <p>
1257     * This implementation avoids any race conditions between scheduling, firing, and cancelling.
1258     */
1259    class Timer {
1260
1261        private final Logger log = RaftKVDatabase.this.log;
1262        private final String name;
1263        private final Service service;
1264        private ScheduledFuture<?> future;
1265        private PendingTimeout pendingTimeout;                  // non-null IFF timeout has not been handled yet
1266        private Timestamp timeoutDeadline;
1267
1268        public Timer(String name, Service service) {
1269            this.name = name;
1270            this.service = service;
1271        }
1272
1273        /**
1274         * Stop timer if running.
1275         *
1276         * @throws IllegalStateException if the lock object is not locked
1277         */
1278        public void cancel() {
1279
1280            // Sanity check
1281            assert Thread.holdsLock(RaftKVDatabase.this);
1282
1283            // Cancel existing timer, if any
1284            if (this.future != null) {
1285                this.future.cancel(false);
1286                this.future = null;
1287            }
1288
1289            // Ensure the previously scheduled action does nothing if case we lose the cancel() race condition
1290            this.pendingTimeout = null;
1291            this.timeoutDeadline = null;
1292        }
1293
1294        /**
1295         * (Re)schedule this timer. Discards any previously scheduled timeout.
1296         *
1297         * @param delay delay before expiration in milliseonds
1298         * @return true if restarted, false if executor rejected the task
1299         * @throws IllegalStateException if the lock object is not locked
1300         */
1301        public void timeoutAfter(int delay) {
1302
1303            // Sanity check
1304            assert Thread.holdsLock(RaftKVDatabase.this);
1305            Preconditions.checkArgument(delay >= 0, "delay < 0");
1306
1307            // Cancel existing timeout action, if any
1308            this.cancel();
1309            assert this.future == null;
1310            assert this.pendingTimeout == null;
1311            assert this.timeoutDeadline == null;
1312
1313            // Reschedule new timeout action
1314            this.timeoutDeadline = new Timestamp().offset(delay);
1315            if (this.log.isTraceEnabled()) {
1316                RaftKVDatabase.this.trace("rescheduling " + this.name + " for " + this.timeoutDeadline
1317                  + " (" + delay + "ms from now)");
1318            }
1319            this.pendingTimeout = new PendingTimeout();
1320            try {
1321                this.future = RaftKVDatabase.this.executor.schedule(this.pendingTimeout, delay, TimeUnit.MILLISECONDS);
1322            } catch (RejectedExecutionException e) {
1323                if (!RaftKVDatabase.this.shuttingDown)
1324                    RaftKVDatabase.this.warn("can't restart timer", e);
1325            }
1326        }
1327
1328        /**
1329         * Force timer to expire immediately.
1330         */
1331        public void timeoutNow() {
1332            this.timeoutAfter(0);
1333        }
1334
1335        /**
1336         * Determine if this timer has expired and requires service handling, and reset it if so.
1337         *
1338         * <p>
1339         * If this timer is not running, has not yet expired, or has previously expired and this method was already
1340         * thereafter invoked, false is returned. Otherwise, true is returned, this timer is {@link #cancel}ed (if necessary),
1341         * and the caller is expected to handle the implied service need.
1342         *
1343         * @return true if timer needs handling, false otherwise
1344         */
1345        public boolean pollForTimeout() {
1346
1347            // Sanity check
1348            assert Thread.holdsLock(RaftKVDatabase.this);
1349
1350            // Has timer expired?
1351            if (this.pendingTimeout == null || !this.timeoutDeadline.hasOccurred())
1352                return false;
1353
1354            // Yes, timer requires service
1355            if (Timer.this.log.isTraceEnabled())
1356                RaftKVDatabase.this.trace(Timer.this.name + " expired " + -this.timeoutDeadline.offsetFromNow() + "ms ago");
1357            this.cancel();
1358            return true;
1359        }
1360
1361        /**
1362         * Determine if this timer is running, i.e., will expire or has expired but
1363         * {@link #pollForTimeout} has not been invoked yet.
1364         */
1365        public boolean isRunning() {
1366            return this.pendingTimeout != null;
1367        }
1368
1369        private class PendingTimeout extends ErrorLoggingRunnable {
1370
1371            @Override
1372            protected void doRun() {
1373                synchronized (RaftKVDatabase.this) {
1374
1375                    // Avoid cancel() race condition
1376                    if (Timer.this.pendingTimeout != this)
1377                        return;
1378
1379                    // Trigger service
1380                    RaftKVDatabase.this.requestService(Timer.this.service);
1381                }
1382            }
1383        }
1384    }

1751    private boolean sendMessage(Message msg) {
1752
1753        // Sanity check
1754        assert Thread.holdsLock(this);
1755
1756        // Get peer's address; if unknown, use the return address of the message being processed (if any)
1757        final String peer = msg.getRecipientId();
1758        String address = this.currentConfig.get(peer);
1759        if (address == null)

1761        if (address == null) {

1764        }
1765

1767        if (this.log.isTraceEnabled())
1768            this.trace("XMIT " + msg + " to " + address);

1770            this.transmitting.add(address);
1771            return true;

1775        this.warn("transmit of " + msg + " to \"" + peer + "\" failed locally");
1776        return false;

1779    private synchronized void receiveMessage(String address, Message msg) {
1780
1781        // Sanity check
1782        assert Thread.holdsLock(this);
1783        assert this.checkState();
1784        if (this.role == null) {
1785            if (this.log.isDebugEnabled())
1786                this.debug("rec'd " + msg + " rec'd in shutdown state; ignoring");
1787            return;
1788        }
1789
1790        // Sanity check cluster ID
1791        if (msg.getClusterId() == 0) {
1792            this.warn("rec'd " + msg + " with zero cluster ID from " + address + "; ignoring");
1793            return;
1794        }
1795        if (this.clusterId != 0 && msg.getClusterId() != this.clusterId) {
1796            this.warn("rec'd " + msg + " with foreign cluster ID "
1797              + String.format("0x%08x", msg.getClusterId()) + " != " + String.format("0x%08x", this.clusterId) + "; ignoring");
1798            return;
1799        }
1800
1801        // Sanity check sender
1802        final String peer = msg.getSenderId();
1803        if (peer.equals(this.identity)) {
1804            this.warn("rec'd " + msg + " from myself (\"" + peer + "\", address " + address + "); ignoring");
1805            return;
1806        }
1807
1808        // Sanity check recipient
1809        final String dest = msg.getRecipientId();
1810        if (!dest.equals(this.identity)) {
1811            this.warn("rec'd misdirected " + msg + " intended for \"" + dest + "\" from " + address + "; ignoring");
1812            return;
1813        }
1814
1815        // Is sender's term too low? Ignore it
1816        if (msg.getTerm() < this.currentTerm) {
1817            this.info("rec'd " + msg + " with term " + msg.getTerm() + " < " + this.currentTerm + " from \""
1818              + peer + "\" at " + address + ", ignoring");
1819            return;
1820        }
1821
1822        // Is my term too low? If so update and revert to follower
1823        if (msg.getTerm() > this.currentTerm) {
1824
1825            // First check with current role; in some special cases we ignore this
1826            if (!this.role.mayAdvanceCurrentTerm(msg)) {
1827                if (this.log.isTraceEnabled()) {
1828                    this.trace("rec'd " + msg + " with term " + msg.getTerm() + " > " + this.currentTerm + " from \""
1829                      + peer + "\" but current role says to ignore it");
1830                }
1831                return;
1832            }
1833
1834            // Revert to follower
1835            this.info("rec'd " + msg.getClass().getSimpleName() + " with term " + msg.getTerm() + " > " + this.currentTerm
1836              + " from \"" + peer + "\", updating term and "
1837              + (this.role instanceof FollowerRole ? "remaining a" : "reverting to") + " follower");
1838            if (!this.advanceTerm(msg.getTerm()))
1839                return;
1840            this.changeRole(msg.isLeaderMessage() ? new FollowerRole(this, peer, address) : new FollowerRole(this));
1841        }
1842
1843        // Debug
1844        if (this.log.isTraceEnabled())
1845            this.trace("RECV " + msg + " in " + this.role + " from " + address);
1846
1847        // Handle message
1848        this.returnAddress = address;
1849        try {
1850            msg.visit(this.role);
1851        } finally {
1852            this.returnAddress = null;
1853        }
1854    }

1895    private Map<String, String> decodeConfig(byte[] key) throws IOException {

2539         *  <li>After a log entry that contains a configuration change has been added to the log</li>
2540         *  <li>After a follower's {@linkplain Follower#getMatchIndex match index} has advanced</li>
2541         * </ul>
2542         */

2544
2545            // Update my commit index based on when a majority of cluster members have ack'd log entries
2546            final long lastLogIndex = this.raft.getLastLogIndex();
2547            while (this.raft.commitIndex < lastLogIndex) {
2548
2549                // Get the index in question
2550                final long index = this.raft.commitIndex + 1;
2551
2552                // Count the number of nodes which have a copy of the log entry at index
2553                int numVotes = this.raft.isClusterMember() ? 1 : 0;                         // count myself, if member
2554                for (Follower follower : this.followerMap.values()) {                       // count followers who are members
2555                    if (follower.getMatchIndex() >= index && this.raft.isClusterMember(follower.getIdentity()))
2556                        numVotes++;
2557                }
2558
2559                // Do a majority of cluster nodes have a copy of this log entry?
2560                final int majority = this.raft.currentConfig.size() / 2 + 1;
2561                if (numVotes < majority)

2593        /**
2594         * Update my {@code leaseTimeout} based on followers' returned {@code leaderTimeout}'s.
2595         *
2596         * <p>
2597         * This should be invoked:
2598         * <ul>
2599         *  <li>After a follower has replied with an {@link AppendResponse} containing a newer
2600         *      {@linkplain AppendResponse#getLeaderTimestamp leader timestamp} than before</li>
2601         * </ul>
2602         */
2603        private void updateLeaseTimeout() {
2604
2605            // Only needed when we have followers
2606            final int numFollowers = this.followerMap.size();
2607            if (numFollowers == 0)
2608                return;
2609
2610            // Get all cluster member leader timestamps, sorted in increasing order
2611            final Timestamp[] leaderTimestamps = new Timestamp[this.raft.currentConfig.size()];
2612            int index = 0;
2613            if (this.raft.isClusterMember())
2614                leaderTimestamps[index++] = new Timestamp();                                // only matters in single node cluster
2615            for (Follower follower : this.followerMap.values()) {
2616                if (this.raft.isClusterMember(follower.getIdentity()))
2617                    leaderTimestamps[index++] = follower.getLeaderTimestamp();
2618            }
2619            Preconditions.checkArgument(index == leaderTimestamps.length);                  // sanity check
2620            Arrays.sort(leaderTimestamps);
2621
2622            //
2623            // Calculate highest leaderTimeout shared by a majority of cluster members, based on sorted array:
2624            //
2625            //  # nodes    timestamps
2626            //  -------    ----------
2627            //     5       [ ][ ][x][x][x]        3/5 x's make a majority at index (5 - 1)/2 = 2
2628            //     6       [ ][ ][x][x][x][x]     4/6 x's make a majority at index (6 - 1)/2 = 2
2629            //
2630            // The minimum leaderTimeout shared by a majority of nodes is at index (leaderTimestamps.length - 1) / 2.
2631            // We then add the minimum election timeout, then subtract a little for clock drift.
2632            //
2633            final Timestamp newLeaseTimeout = leaderTimestamps[(leaderTimestamps.length + 1) / 2]
2634              .offset((int)(this.raft.minElectionTimeout * (1.0f - MAX_CLOCK_DRIFT)));
2635
2636            // Immediately notify any followers who are waiting on this updated timestamp (or any smaller timestamp)
2637            if (newLeaseTimeout.compareTo(this.leaseTimeout) > 0) {
2638
2639                // Update my leader lease timeout
2640                if (this.log.isTraceEnabled())
2641                    this.trace("updating my lease timeout from " + this.leaseTimeout + " -> " + newLeaseTimeout);
2642                this.leaseTimeout = newLeaseTimeout;
2643
2644                // Notify any followers who care
2645                for (Follower follower : this.followerMap.values()) {
2646                    final NavigableSet<Timestamp> timeouts = follower.getCommitLeaseTimeouts().headSet(this.leaseTimeout, true);
2647                    if (!timeouts.isEmpty()) {
2648                        follower.updateNow();                           // notify follower so it can commit waiting transaction(s)
2649                        timeouts.clear();
2650                    }
2651                }
2652            }
2653        }

2715        /**
2716         * Check whether a follower needs an update and send one if so.
2717         *
2718         * <p>
2719         * This should be invoked:
2720         * <ul>
2721         *  <li>After a new follower has been added</li>
2722         *  <li>When the output queue for a follower goes from non-empty to empty</li>
2723         *  <li>After the follower's {@linkplain Follower#getUpdateTimer update timer} has expired</li>
2724         *  <li>After a new log entry has been added to the log (all followers)</li>
2725         *  <li>After receiving an {@link AppendResponse} that caused the follower's
2726         *      {@linkplain Follower#getNextIndex next index} to change</li>
2727         *  <li>After receiving the first positive {@link AppendResponse} to a probe</li>
2728         *  <li>After our {@code commitIndex} has advanced (all followers)</li>
2729         *  <li>After our {@code leaseTimeout} has advanced past one or more of a follower's
2730         *      {@linkplain Follower#getCommitLeaseTimeouts commit lease timeouts} (with update timer reset)</li>
2731         *  <li>After sending a {@link CommitResponse} with a non-null {@linkplain CommitResponse#getCommitLeaderLeaseTimeout
2732         *      commit leader lease timeout} (all followers) to probe for updated leader timestamps</li>
2733         *  <li>After starting, aborting, or completing a snapshot install for a follower</li>
2734         * </ul>
2735         */
2736        private void updateFollower(Follower follower) {
2737
2738            // If follower has an in-progress snapshot that has become too stale, abort it
2739            final String peer = follower.getIdentity();
2740            SnapshotTransmit snapshotTransmit = follower.getSnapshotTransmit();
2741            if (snapshotTransmit != null && snapshotTransmit.getSnapshotIndex() < this.raft.lastAppliedIndex) {
2742                if (this.log.isDebugEnabled())
2743                    this.debug("aborting stale snapshot install for " + follower);
2744                follower.cancelSnapshotTransmit();
2745                follower.updateNow();
2746            }
2747
2748            // Is follower's queue empty? If not, hold off until then
2749            if (this.raft.isTransmitting(follower.getAddress())) {
2750                if (this.log.isTraceEnabled())
2751                    this.trace("no update for \"" + peer + "\": output queue still not empty");
2752                return;
2753            }
2754
2755            // Handle any in-progress snapshot install
2756            if ((snapshotTransmit = follower.getSnapshotTransmit()) != null) {
2757
2758                // Send the next chunk in transmission, if any
2759                final long pairIndex = snapshotTransmit.getPairIndex();
2760                final ByteBuffer chunk = snapshotTransmit.getNextChunk();
2761                boolean synced = true;
2762                if (chunk != null) {
2763
2764                    // Send next chunk
2765                    final InstallSnapshot msg = new InstallSnapshot(this.raft.clusterId, this.raft.identity, peer,
2766                      this.raft.currentTerm, snapshotTransmit.getSnapshotTerm(), snapshotTransmit.getSnapshotIndex(), pairIndex,
2767                      pairIndex == 0 ? snapshotTransmit.getSnapshotConfig() : null, !snapshotTransmit.hasMoreChunks(), chunk);
2768                    if (this.raft.sendMessage(msg))
2769                        return;
2770                    if (this.log.isDebugEnabled())
2771                        this.debug("canceling snapshot install for " + follower + " due to failure to send " + msg);
2772
2773                    // Message failed -> snapshot is fatally wounded, so cancel it
2774                    synced = false;
2775                }
2776                if (synced)
2777                    this.info("completed snapshot install for out-of-date " + follower);
2778
2779                // Snapshot transmit is complete (or failed)
2780                follower.cancelSnapshotTransmit();
2781
2782                // Trigger an immediate regular update
2783                follower.setNextIndex(snapshotTransmit.getSnapshotIndex() + 1);
2784                follower.setSynced(synced);
2785                follower.updateNow();
2786                this.raft.requestService(new UpdateFollowerService(follower));
2787                return;
2788            }
2789
2790            // Are we still waiting for the update timer to expire?
2791            if (!follower.getUpdateTimer().pollForTimeout()) {
2792                boolean waitForTimerToExpire = true;
2793
2794                // Don't wait for the update timer to expire if:
2795                //  (a) The follower is sync'd; AND
2796                //      (y) We have a new log entry that the follower doesn't have; OR
2797                //      (y) We have a new leaderCommit that the follower doesn't have
2798                // The effect is that we will pipeline updates to synchronized followers.
2799                if (follower.isSynced()
2800                  && (follower.getLeaderCommit() != this.raft.commitIndex
2801                   || follower.getNextIndex() <= this.raft.getLastLogIndex()))
2802                    waitForTimerToExpire = false;
2803
2804                // Wait for timer to expire
2805                if (waitForTimerToExpire) {
2806                    if (this.log.isTraceEnabled()) {
2807                        this.trace("no update for \"" + follower.getIdentity() + "\": timer not expired yet, and follower is "
2808                          + (follower.isSynced() ? "up to date" : "not synced"));
2809                    }
2810                    return;
2811                }
2812            }
2813
2814            // Get index of the next log entry to send to follower
2815            final long nextIndex = follower.getNextIndex();
2816
2817            // If follower is too far behind, we must do a snapshot install
2818            if (nextIndex <= this.raft.lastAppliedIndex) {
2819                final MostRecentView view = this.raft.new MostRecentView();
2820                follower.setSnapshotTransmit(new SnapshotTransmit(this.raft.getLastLogTerm(), this.raft.getLastLogIndex(),
2821                  this.raft.lastAppliedConfig, view.getSnapshot(), view.getView()));
2822                this.info("started snapshot install for out-of-date " + follower);
2823                this.raft.requestService(new UpdateFollowerService(follower));
2824                return;
2825            }
2826
2827            // Restart update timer here (to avoid looping if an error occurs below)
2828            follower.getUpdateTimer().timeoutAfter(this.raft.heartbeatTimeout);
2829
2830            // Send actual data if follower is synced and there is a log entry to send; otherwise, just send a probe
2831            final AppendRequest msg;
2832            if (!follower.isSynced() || nextIndex > this.raft.getLastLogIndex()) {
2833                msg = new AppendRequest(this.raft.clusterId, this.raft.identity, peer, this.raft.currentTerm, new Timestamp(),
2834                  this.leaseTimeout, this.raft.commitIndex, this.raft.getLogTermAtIndex(nextIndex - 1), nextIndex - 1);   // probe
2835            } else {
2836
2837                // Get log entry to send
2838                final LogEntry logEntry = this.raft.getLogEntryAtIndex(nextIndex);
2839
2840                // If the log entry correspond's to follower's transaction, don't send the data because follower already has it.
2841                // But only do this optimization the first time, in case something goes wrong on the follower's end.
2842                ByteBuffer mutationData = null;
2843                if (!follower.getSkipDataLogEntries().remove(logEntry)) {
2844                    try {
2845                        mutationData = logEntry.getContent();
2846                    } catch (IOException e) {
2847                        this.error("error reading log file " + logEntry.getFile(), e);
2848                        return;
2849                    }
2850                }
2851
2852                // Create message
2853                msg = new AppendRequest(this.raft.clusterId, this.raft.identity, peer, this.raft.currentTerm, new Timestamp(),
2854                  this.leaseTimeout, this.raft.commitIndex, this.raft.getLogTermAtIndex(nextIndex - 1), nextIndex - 1,
2855                  logEntry.getTerm(), mutationData);
2856            }
2857
2858            // Send update
2859            final boolean sent = this.raft.sendMessage(msg);
2860
2861            // Advance next index if a log entry was sent; we allow pipelining log entries when synchronized
2862            if (sent && !msg.isProbe()) {
2863                assert follower.isSynced();
2864                follower.setNextIndex(Math.min(follower.getNextIndex(), this.raft.getLastLogIndex()) + 1);
2865            }
2866
2867            // Update leaderCommit for follower
2868            if (sent)
2869                follower.setLeaderCommit(msg.getLeaderCommit());
2870        }
